To Set a Transaction Timeout Value

By default, the server does not time out a transaction. That is, the server waits indefinitely for a transaction to complete. If you set a timeout value for transactions, if a transaction isn’t completed within the configured time, the Enterprise Server rolls back the transaction.

  1. In the navigation tree, expand the Configuration node.
  2. Select the Transaction Service node.

    The Transaction Service page opens.

  3. In the Transaction Timeout field, type the number of seconds before the transaction times out.

    The default value is 0. This value disables transaction timeouts.

  4. Click Save.
  5. Restart the Enterprise Server.
