To Create an Administered Object Resource

  1. In the navigation tree, expand the Resources node.
  2. Expand the Connectors node
  3. Under the Connectors node, select the Admin Object Resources node.

    The Admin Object Resources page opens.

  4. On the Admin Object Resources page, click the New button.

    The New Admin Object Resource page opens.

  5. In the JNDI Name field, type a unique name that identifies the administered object resource.
  6. From the Resource Adapter drop-down list, select the resource adapter for the administered object resource.
  7. From the Resource Type drop-down list, select the Java type for the administered object resource.
  8. From the Class Name drop-down list, select the implementation class name associated with the resource type.
  9. In the Description field, type a description of the administered object resource.
  10. Select the Status Enabled checkbox to enable the administered object resource.

    This option is enabled by default.

  11. (Optional) In the Additional Properties table, specify additional properties for the administered object resource.

    Available properties depend upon the resource adapter for the connector connection pool.

    To add a property, click the Add Property button. In the blank row that appears, type the property name in the Name field, and type the property value in the Value field.

  12. In the Targets Available section, select the domain, cluster, or server instances where the administered object will reside and click Add.

    To remove one of the domains, clusters, or server instances listed in the Selected field, select it from the field and click Remove.

  13. Click OK.
