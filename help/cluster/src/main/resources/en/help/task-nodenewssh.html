To Finish the Creation of an SSH Node

This task is the continuation for an SSH node of To Create a Node.

  1. To copy GlassFish Server software from the DAS host to the node host, enable the Install GlassFish Server option.

  2. To force the node to be created in the configuration of the domain administration server (DAS) even if validation of the node's parameters fails, enable the Force option.

  3. In the SSH Port field, type the port to use for SSH connections to this node's host.

  4. In the SSH User Name field, type the name of the user on this node's host that is to run the process for connecting to the host through SSH.

  5. Specify how the SSH user will be authenticated when logging in to this node's host.

    • To specify authentication through an SSH private key file, perform these steps:

      1. From the SSH User Authentication drop-down list, select Key File.

      2. If you require GlassFish Server to set up an SSH key on the node host, enable the Setup SSH Option.

        GlassFish Server sets up the SSH key by copying existing key files from the DAS host or, if required, by generating the files before copying the files.

        The Generate Key field and the Public Key File field are added to the window.

      3. If you require GlassFish Server to generate the SSH key files before copying the files to the node host, enable the Generate Key Option.

      4. In the SSH User Password field, type the password that the SSH user will use when logging in to this node's host.

      5. In the Public Key File field, type the absolute path to the SSH public key file for user that the SSH User Name option specifies.

      6. In the Key File field, type the absolute path to the SSH private key file for user that the SSH User Name option specifies.


        Note:

        GlassFish Server also supports password authentication through the AS_ADMIN_SSHPASSWORD entry in the password file. The password file is specified in the --paswordfile option of the asadmin(1) utility.


    • To specify authentication through a password that is stored in the DAS configuration, perform these steps:


      Caution:

      This password is stored in clear text in the DAS configuration. For enhanced security, use a key file or a password alias


      1. From the SSH User Authentication drop-down list, select Password.

      2. In the SSH User Password field, type the password that the SSH user will use when logging in to this node's host.

      3. In the Confirm Password field, type the password again.

    • To specify authentication through a password alias, perform these steps:

      1. From the SSH User Authentication drop-down list, select Password Alias.

      2. From the Password Alias drop-down list, select an existing alias.

  6. Click Save.
