To Delete Clustered GlassFish Server Instances

Caution - If you are using a Java Message Service (JMS) cluster with a master broker, do not delete the instance that is associated with the master broker. If this instance must be deleted, use the change-master-broker(1) subcommand to assign the master broker to a different instance.

  1. In the navigation tree, expand the Clusters node.
  2. Under the expanded Clusters node, select the node for the cluster.

    The General Information page for the cluster opens.

  3. On the General Information page, click the Instances tab.

    The Clustered Server Instances page for the cluster opens.

  4. For each instance that you are deleting, select the check box adjacent to the instance in the Server Instances table.
  5. Click Delete.

    You are asked to confirm that you want to delete the instances.

  6. Confirm that you want to delete the instances.

    The instances are deleted and are removed from the table of instances on the page.
