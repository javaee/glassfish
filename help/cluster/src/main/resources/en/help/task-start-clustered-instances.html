To Start Clustered GlassFish Server Instances

Before You Begin

Ensure that following prerequisites are met:

  • All nodes where the instances reside are enabled for remote communication.

  • The user of the domain administration server (DAS) can use secure shell (SSH) to log in to the host for any node where the instances reside.

If any of these prerequisites is not met, start each instance individually from the command line by using the start-local-instance(1) subcommand.

  1. In the navigation tree, expand the Clusters node.
  2. Under the expanded Clusters node, select the node for the cluster.

    The General Information page for the cluster opens.

  3. On the General Information page, click the Instances tab.

    The Clustered Server Instances page for the cluster opens.

  4. For each instance that you are starting, select the check box adjacent to the instance in the Server Instances table.
  5. Click Start.

    You are asked to confirm that you want to start the instances.

  6. Confirm that you want to start the instances.

    The status of each instance is updated in the Server Instances table when the instance is started.
