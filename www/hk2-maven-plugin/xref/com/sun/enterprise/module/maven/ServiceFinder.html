59  /**
60   * A simple service-provider lookup mechanism.  A <i>service</i> is a
61   * well-known set of interfaces and (usually abstract) classes.  A <i>service
62   * provider</i> is a specific implementation of a service.  The classes in a
63   * provider typically implement the interfaces and subclass the classes defined
64   * in the service itself.  Service providers may be installed in an
65   * implementation of the Java platform in the form of extensions, that is, jar
66   * files placed into any of the usual extension directories.  Providers may
67   * also be made available by adding them to the applet or application class
68   * path or by some other platform-specific means.
69   * <p/>
70   * <p> In this lookup mechanism a service is represented by an interface or an
71   * abstract class.  (A concrete class may be used, but this is not
72   * recommended.)  A provider of a given service contains one or more concrete
73   * classes that extend this <i>service class</i> with data and code specific to
74   * the provider.  This <i>provider class</i> will typically not be the entire
75   * provider itself but rather a proxy that contains enough information to
76   * decide whether the provider is able to satisfy a particular request together
77   * with code that can create the actual provider on demand.  The details of
78   * provider classes tend to be highly service-specific; no single class or
79   * interface could possibly unify them, so no such class has been defined.  The
80   * only requirement enforced here is that provider classes must have a
81   * zero-argument constructor so that they may be instantiated during lookup.
82   * <p/>
83   * <p> A service provider identifies itself by placing a provider-configuration
84   * file in the resource directory <tt>META-INF/services</tt>.  The file's name
85   * should consist of the fully-qualified name of the abstract service class.
86   * The file should contain a list of fully-qualified concrete provider-class
87   * names, one per line.  Space and tab characters surrounding each name, as
88   * well as blank lines, are ignored.  The comment character is <tt>'#'</tt>
89   * (<tt>0x23</tt>); on each line all characters following the first comment
90   * character are ignored.  The file must be encoded in UTF-8.
91   * <p/>
92   * <p> If a particular concrete provider class is named in more than one
93   * configuration file, or is named in the same configuration file more than
94   * once, then the duplicates will be ignored.  The configuration file naming a
95   * particular provider need not be in the same jar file or other distribution
96   * unit as the provider itself.  The provider must be accessible from the same
97   * class loader that was initially queried to locate the configuration file;
98   * note that this is not necessarily the class loader that found the file.
99   * <p/>
100  * <p> <b>Example:</b> Suppose we have a service class named
101  * <tt>java.io.spi.CharCodec</tt>.  It has two abstract methods:
102  * <p/>
103  * <pre>
104  *   public abstract CharEncoder getEncoder(String encodingName);
105  *   public abstract CharDecoder getDecoder(String encodingName);
106  * </pre>
107  * <p/>
108  * Each method returns an appropriate object or <tt>null</tt> if it cannot
109  * translate the given encoding.  Typical <tt>CharCodec</tt> providers will
110  * support more than one encoding.
111  * <p/>
112  * <p> If <tt>sun.io.StandardCodec</tt> is a provider of the <tt>CharCodec</tt>
113  * service then its jar file would contain the file
114  * <tt>META-INF/services/java.io.spi.CharCodec</tt>.  This file would contain
115  * the single line:
116  * <p/>
117  * <pre>
118  *   sun.io.StandardCodec    # Standard codecs for the platform
119  * </pre>
120  * <p/>
121  * To locate an codec for a given encoding name, the internal I/O code would
122  * do something like this:
123  * <p/>
124  * <pre>
125  *   CharEncoder getEncoder(String encodingName) {
126  *       for( CharCodec cc : ServiceFinder.find(CharCodec.class) ) {
127  *           CharEncoder ce = cc.getEncoder(encodingName);
128  *           if (ce != null)
129  *               return ce;
130  *       }
131  *       return null;
132  *   }
133  * </pre>
134  * <p/>
135  * The provider-lookup mechanism always executes in the security context of the
136  * caller.  Trusted system code should typically invoke the methods in this
137  * class from within a privileged security context.
138  *
139  * @author Mark Reinhold
140  * @version 1.11, 03/12/19
141  * @since 1.3
142  */
143 final class ServiceFinder<T> implements Iterable<T> {

145     private static final String prefix = "META-INF/services/";
146 
147     private final Class<T> serviceClass;
148     private final @Nullable
149     ClassLoader classLoader;

151     /**
152      * Locates and incrementally instantiates the available providers of a
153      * given service using the given class loader.
154      * <p/>
155      * <p> This method transforms the name of the given service class into a
156      * provider-configuration filename as described above and then uses the
157      * <tt>getResources</tt> method of the given class loader to find all
158      * available files with that name.  These files are then read and parsed to
159      * produce a list of provider-class names.  The iterator that is returned
160      * uses the given class loader to lookup and then instantiate each element
161      * of the list.
162      * <p/>
163      * <p> Because it is possible for extensions to be installed into a running
164      * Java virtual machine, this method may return different results each time
165      * it is invoked. <p>
166      *
167      * @param service The service's abstract service class
168      * @param loader  The class loader to be used to load provider-configuration files
169      *                and instantiate provider classes, or <tt>null</tt> if the system
170      *                class loader (or, failing that the bootstrap class loader) is to
171      *                be used
172      * @throws ServiceConfigurationError If a provider-configuration file violates the specified format
173      *                                   or names a provider class that cannot be found and instantiated
174      * @see #find(Class)
175      */
176     public static <T> ServiceFinder<T> find(@NotNull Class<T> service, @Nullable ClassLoader loader) {
177         return new ServiceFinder<T>(service,loader);
178     }

180     /**
181      * Locates and incrementally instantiates the available providers of a
182      * given service using the context class loader.  This convenience method
183      * is equivalent to
184      * <p/>
185      * <pre>
186      *   ClassLoader cl = Thread.currentThread().getContextClassLoader();
187      *   return Service.providers(service, cl);
188      * </pre>
189      *
190      * @param service The service's abstract service class
191      *
192      * @throws ServiceConfigurationError If a provider-configuration file violates the specified format
193      *                                   or names a provider class that cannot be found and instantiated
194      * @see #find(Class, ClassLoader)
195      */
196     public static <T> ServiceFinder<T> find(Class<T> service) {
197         return find(service,Thread.currentThread().getContextClassLoader());
198     }

200     private ServiceFinder(Class<T> service, ClassLoader loader) {
201         this.serviceClass = service;
202         this.classLoader = loader;

205     /**
206      * Returns discovered objects incrementally.
207      *
208      * @return An <tt>Iterator</tt> that yields provider objects for the given
209      *         service, in some arbitrary order.  The iterator will throw a
210      *         <tt>ServiceConfigurationError</tt> if a provider-configuration
211      *         file violates the specified format or if a provider class cannot

218     /**
219      * Returns discovered objects all at once.
220      *
221      * @return
222      *      can be empty but never null.

224      * @throws ServiceConfigurationError

226     public T[] toArray() {
227         List<T> result = new ArrayList<T>();
228         for (T t : this) {
229             result.add(t);

252     /**
253      * Parse a single line from the given configuration file, adding the name
254      * on the line to both the names list and the returned set iff the name is
255      * not already a member of the returned set.
256      */
257     private static int parseLine(Class service, URL u, BufferedReader r, int lc,
258                                  List<String> names, Set<String> returned)
259         throws IOException, ServiceConfigurationError {
260         String ln = r.readLine();
261         if (ln == null) {
262             return -1;
263         }
264         int ci = ln.indexOf('#');
265         if (ci >= 0) ln = ln.substring(0, ci);
266         ln = ln.trim();
267         int n = ln.length();
268         if (n != 0) {
269             if ((ln.indexOf(' ') >= 0) || (ln.indexOf('\t') >= 0))
270                 fail(service, u, lc, "Illegal configuration-file syntax");
271             int cp = ln.codePointAt(0);
272             if (!Character.isJavaIdentifierStart(cp))
273                 fail(service, u, lc, "Illegal provider-class name: " + ln);
274             for (int i = Character.charCount(cp); i < n; i += Character.charCount(cp)) {
275                 cp = ln.codePointAt(i);
276                 if (!Character.isJavaIdentifierPart(cp) && (cp != '.'))
277                     fail(service, u, lc, "Illegal provider-class name: " + ln);
278             }
279             if (!returned.contains(ln)) {
280                 names.add(ln);
281                 returned.add(ln);
282             }
283         }
284         return lc + 1;
285     }

287     /**
288      * Parse the content of the given URL as a provider-configuration file.
289      *
290      * @param service  The service class for which providers are being sought;
291      *                 used to construct error detail strings
292      * @param u        The URL naming the configuration file to be parsed
293      * @param returned A Set containing the names of provider classes that have already
294      *                 been returned.  This set will be updated to contain the names
295      *                 that will be yielded from the returned <tt>Iterator</tt>.
296      * @return A (possibly empty) <tt>Iterator</tt> that will yield the
297      *         provider-class names in the given configuration file that are
298      *         not yet members of the returned set
299      * @throws ServiceConfigurationError If an I/O error occurs while reading from the given URL, or
300      *                                   if a configuration-file format error is detected
301      */
302     @SuppressWarnings({"StatementWithEmptyBody"})
303     private static Iterator<String> parse(Class service, URL u, Set<String> returned)
304         throws ServiceConfigurationError {
305         InputStream in = null;
306         BufferedReader r = null;
307         ArrayList<String> names = new ArrayList<String>();
308         try {
309             in = u.openStream();
310             r = new BufferedReader(new InputStreamReader(in, "utf-8"));
311             int lc = 1;
312             while ((lc = parseLine(service, u, r, lc, names, returned)) >= 0) ;
313         } catch (IOException x) {
314             fail(service, ": " + x);
315         } finally {
316             try {
317                 if (r != null) r.close();
318                 if (in != null) in.close();
319             } catch (IOException y) {
320                 fail(service, ": " + y);
321             }
322         }
323         return names.iterator();
324     }

327     /**
328      * Private inner class implementing fully-lazy provider lookup
329      */
330     private static class LazyIterator<T> implements Iterator<T> {
331         Class<T> service;
332         @Nullable ClassLoader loader;
333         Enumeration<URL> configs = null;
334         Iterator<String> pending = null;
335         Set<String> returned = new TreeSet<String>();
336         String nextName = null;
337         URL currentConfig = null;
338 
339         private LazyIterator(Class<T> service, ClassLoader loader) {
340             this.service = service;
341             this.loader = loader;
342         }
343 
344         public boolean hasNext() throws ServiceConfigurationError {
345             if (nextName != null) {
346                 return true;
347             }
348             if (configs == null) {
349                 try {
350                     String fullName = prefix + service.getName();
351                     if (loader == null)
352                         configs = ClassLoader.getSystemResources(fullName);
353                     else
354                         configs = loader.getResources(fullName);
355                 } catch (IOException x) {
356                     fail(service, ": " + x);
357                 }
358             }
359             while ((pending == null) || !pending.hasNext()) {
360                 if (!configs.hasMoreElements()) {
361                     return false;
362                 }
363                 currentConfig = configs.nextElement();
364                 pending = parse(service, currentConfig, returned);
365             }
366             nextName = pending.next();
367             return true;
368         }
369 
370         public T next() throws ServiceConfigurationError {
371             if (!hasNext()) {
372                 throw new NoSuchElementException();
373             }
374             String cn = nextName;
375             nextName = null;
376             try {
377                 return service.cast(Class.forName(cn, true, loader).newInstance());
378             } catch (ClassNotFoundException x) {
379                 fail(service,
380                     "Provider " + cn + " is specified in "+currentConfig+" but not found");
381             } catch (Exception x) {
382                 fail(service,
383                     "Provider " + cn + " is specified in "+currentConfig+"but could not be instantiated: " + x, x);
384             }
385             return null;    /* This cannot happen */
386         }
387 
388         public void remove() {
389             throw new UnsupportedOperationException();
390         }
391     }
