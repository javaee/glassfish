76  /**
77   * The default {@link org.glassfish.hk2.runlevel.RunLevelController} implementation for Hk2. See the
78   * {@link org.glassfish.hk2.runlevel.RunLevelController} javadoc for general details regarding this service.
79   * <p/>
80   * Here is a brief example of the behavior of this service:<br>
81   * <p/>
82   * Imagine services ServiceA, ServiceB, and ServiceC are all in the same
83   * RunLevel X and the dependencies are ServiceA -> ServiceB -> ServiceC:
84   * <p/>
85   * <code>
86   * &nbsp;@RunLevel(X)<br/>
87   * &nbsp;@Service<br/>
88   * public class ServiceA {<br/>
89   * &nbsp;@Inject ServiceB b;<br/>
90   * }<br/>
91   * <br/>
92   * &nbsp;@RunLevel(X)<br/>
93   * &nbsp;@Service<br/>
94   * public class ServiceB {<br/>
95   * &nbsp;@Inject ServiceC c;<br/>
96   * }<br/>
97   * <br/>
98   * &nbsp;@RunLevel(X)<br/>
99   * &nbsp;@Service<br/>
100  * public class ServiceC {<br/>
101  * }<br/>
102  * </code>
103  * <p/>
104  * When the DefaultRunLevelController is asked to proceedTo(X), the expected start
105  * order is: ServiceC, ServiceB, ServiceA, and the expected shutdown order is:
106  * ServiceA, ServiceB, ServiceC
107  * <p/>
108  * Note that no model of dependencies between services are kept in the habitat
109  * to make the implementation work. Any inhabitant in RunLevel X is arbitrarily
110  * picked to start with upon activation, and {@link ActiveDescriptor#create(org.glassfish.hk2.api.ServiceHandle)} is issued.
111  * <p/>
112  * Consider the cases of possible activation orderings:
113  * <p/>
114  * Case 1: A, B, then C by RLS. get ServiceA (called by RLS) Start ServiceA: get
115  * ServiceB Start ServiceB: get ServiceC Start ServiceC wire ServiceC
116  * PostConstruct ServiceC wire ServiceB PostConstruct ServiceB wire ServiceA
117  * PostConstruct ServiceA get ServiceB (called by RLS) get ServiceC (called by
118  * RLS)
119  * <p/>
120  * Case 2: B, C, then A by RLS. get ServiceB (called by RLS) Start ServiceB: get
121  * ServiceC Start ServiceC wire ServiceC PostConstruct ServiceC wire ServiceB
122  * PostConstruct ServiceB get ServiceC (called by RLS) get ServiceA (called by
123  * RLS) Start ServiceA: get ServiceB wire ServiceA PostConstruct ServiceA
124  * <p/>
125  * Case 3: B, A, then C by RLS. get ServiceB (called by RLS) Start ServiceB: get
126  * ServiceC Start ServiceC wire ServiceC PostConstruct ServiceC wire ServiceB
127  * PostConstruct ServiceB get ServiceA (called by RLS) Start ServiceA: get
128  * ServiceB wire ServiceA PostConstruct ServiceA get ServiceC (called by RLS)
129  * <p/>
130  * Case 4: C, B, then A by RLS. get ServiceC (called by RLS) Start ServiceC:
131  * wire ServiceC PostConstruct ServiceC get ServiceB (called by RLS) Start
132  * ServiceB: get ServiceC wire ServiceB PostConstruct ServiceB get ServiceA
133  * (called by RLS) Start ServiceA: get ServiceB wire ServiceA PostConstruct
134  * ServiceA get ServiceA (called by RLS)
135  * <p/>
136  * You can see that the order is always correct without needing to keep the
137  * model of dependencies.
138  * <p/>
139  * ~~~
140  * <p/>
141  * Note that the implementation performs some level of constraint checking
142  * during injection. For example,
143  * <p/>
144  * - It is an error to have a RunLevel-annotated service at RunLevel X to depend
145  * on (i.e., be injected with) a RunLevel-annotated service at RunLevel Y when Y
146  * > X.
147  * <p/>
148  * - It is an error to have a non-RunLevel-annotated service to depend on a
149  * RunLevel-annotated service at any RunLevel.
150  * <p/>
151  * Note that the implementation does not handle Holder and Collection injection
152  * constraint validations.
153  * <p/>
154  * ~~~
155  * <p/>
156  * The implementation will automatically proceedTo(-1) after the habitat has
157  * been initialized. The value of "-1" is symbolic of the kernel run level.
158  * <p/>
159  * Note that all RunLevel values less than -1 will be ignored.
160  * <p/>
161  * ~~~
162  * <p/>
163  * The implementation is written to support two modes of operation, asynchronous
164  * / threaded, and synchronous / single threaded. The DefaultRunLevelController
165  * implementation mode is pre-configured to be synchronous. The
166  * DefaultRunLevelController is thread safe.
167  * <p/>
168  * In the synchronous mode, calls can be made to proceedTo() to interrupt
169  * processing of any currently executing proceedTo() operation. This might
170  * occur: in another thread, in the {@link org.glassfish.hk2.runlevel.RunLevelListener} handlers, or in a
171  * {@link org.glassfish.hk2.runlevel.RunLevel} annotated service's {@link javax.annotation.PostConstruct} method call.
172  * <p/>
173  * Note, however, that even in synchronous mode the proceedTo() operation may
174  * exhibit asynchronous behavior. This is the case when the caller has two
175  * threads calling proceedTo(), where the second thread is canceling the
176  * operation of the first (perhaps due to timeout of a service's PostConstruct,
177  * etc.). In this case, an interrupt will be sent to the first running thread to
178  * cancel the previous operation, and proceedTo the run level from the second
179  * thread's request. This presumes that the first thread is capable of being
180  * interrupted. In such a situation, the second proceedTo() call returns
181  * immediately and the first proceedTo() is interrupted to continue to the new
182  * runLevel requested from the second thread's interrupt.
183  * <p/>
184  * For this reason, it is strongly advised that {@link InterruptedException} is
185  * not swallowed by services that can be driven by the DefaultRunLevelController in
186  * synchronous mode.
187  * <p/>
188  * proceedTo invocations from a {@link javax.annotation.PostConstruct} callback are discouraged.
189  * Consider using {@link org.glassfish.hk2.runlevel.RunLevelListener} instead.
190  * <p/>
191  * <b>Important Note:</b><br>
192  * The proceedTo() method will throw unchecked exceptions if it detects that
193  * it is being called reentrantly in synchronous mode. Callers should be
194  * careful NOT to swallow exceptions of this type as shown in the following
195  * example:
196  * <p/>
197  * <code>
198  * try {<br/>
199  * &nbsp;rls.proceedTo(x);<br>
200  * } catch (Exception e) {<br>
201  * &nbsp;// swallow exception<br>
202  * }
203  * </code>
204  * <p/>
205  * ~~~
206  * <p/>
207  * All calls to the {@link org.glassfish.hk2.runlevel.RunLevelListener} happens synchronously on the
208  * same thread that caused the Inhabitant to be activated. Therefore,
209  * implementors of this interface should be careful and avoid calling long
210  * operations.
211  * <p/>
212  * ~~~
213  * <p/>
214  *
215  * @author jtrent, tbeerbower
216  */

283     private RunLevelControllerImpl(boolean async) {
284         this.asyncMode = async;
285         if (asyncMode) {
286             // we can't use a singleThreadExecutor because a thread could become
287             // "stuck"
288             exec = Executors.newCachedThreadPool(new ThreadFactory() {
289                 @Override
290                 public Thread newThread(Runnable runnable) {
291                     Thread activeThread = new RunLevelControllerThread(runnable);
292                     synchronized (lock) {
293                         logger.log(Level.FINE, "new thread: {0}", activeThread);
294                     }
295                     return activeThread;
296                 }
297             });
298         } else {
299             this.exec = null;
300         }
301     }

438     private boolean isCancelled(Worker worker) {
439         synchronized (lock) {
440             return (this.worker != worker);
441         }
442     }
443 
444     /**
445      * Called after the proceedTo work is finished. This is akin to a suicide
446      * act.
447      *
448      * @param worker the worker that was performing the work
449      */
450     private void finished(Worker worker) {
451         synchronized (lock) {
452             // ensure that the worker is the "active" one and not some zombie
453             // back to life
454             if (!isCancelled(worker)) {
455                 // it was the "trusted" worker
456                 this.worker = null;
457             }
458         }
459 
460         synchronized (this) {
461             notifyAll();
462         }
463     }
464 
465     private void setCurrent(Worker worker, Integer current) {
466         synchronized (lock) {
467             // ensure that the worker is the "active" one and not some zombie
468             // back to life
469             if (isCancelled(worker)) {
470                 return;
471             } else {
472                 this.currentRunLevel = current;
473             }
474         }
475 
476         // notify listeners that we progressed
477         event(worker, ListenerEvent.PROGRESS, null, false);
478     }

480     protected List<Integer> getRecordersToRelease(int runLevel) {
481         List<Integer> qualifying = new ArrayList<Integer>();
482         synchronized (lock) {
483             for (Map.Entry<Integer, Stack<ActiveDescriptor<?>>> entry : recorders.entrySet()) {
484                 int entryKey = entry.getKey();
485                 if (entryKey >= runLevel) {
486                     qualifying.add(entry.getKey());
487                 }
488             }
489         }
490 
491         // return in order of highest to lowest
492         Collections.sort(qualifying);
493         Collections.reverse(qualifying);
494 
495         return qualifying;
496     }

498     protected void event(Worker worker, ListenerEvent event,
499                          Throwable error, boolean isHardInterrupt) {
500         logger.log(LEVEL, "event {0} - " + getDescription(true), event);
501 
502         if (isCancelled(worker)) {
503             logger.log(LEVEL, "Ignoring this notification!");
504         } else {
505             Interrupt lastInterrupt = null;
506             Collection<RunLevelListener> activeListeners = getListeners();
507             for (RunLevelListener listener : activeListeners) {
508                 try {
509                     if (ListenerEvent.PROGRESS == event) {
510                         listener.onProgress(this);
511                     } else if (ListenerEvent.CANCEL == event) {
512                         listener.onCancelled(this, currentRunLevel,
513                                 isHardInterrupt);
514                     } else {
515                         listener.onError(this, error, true);
516                     }
517                 } catch (Interrupt interrupt) {
518                     lastInterrupt = interrupt;
519                 } catch (Exception e) {
520                     // don't percolate the exception since it may negatively
521                     // impact processing
522                     logger.log(Level.WARNING, "swallowing exception - "
523                             + getDescription(true), new RunLevelException(e));
524                 }
525             }
526 
527             if (lastInterrupt != null) {
528                 throw lastInterrupt;
529             } else {
530                 if (error != null) {
531                     logger.log(LEVEL, "swallowing error - " + error);
532                 }
533             }
534         }
535     }

537     /**
538      * Proceed to the given run level.
539      *
540      * @param runLevel         the run level
541      * @param isHardInterrupt  indicates a hard interrupt
542      */
543     protected void proceedTo(Integer runLevel, boolean isHardInterrupt) {
544         if (null != runLevel && runLevel < RunLevel.RUNLEVEL_VAL_IMMEDIATE) {
545             throw new IllegalArgumentException();
546         }
547 
548         // see if we can interrupt first
549         Worker worker = this.worker;
550         if (worker != null) {
551             if (worker.interrupt(isHardInterrupt, runLevel)) {
552                 return;
553             }
554         }
555 
556         if (runLevel != null) {
557             // if we are here then the interrupt isn't enough and we must create
558             // a new worker
559             synchronized (lock) {
560                 this.worker = worker = asyncMode ?
561                         new AsyncProceedToWorker(runLevel) :
562                         new SyncProceedToWorker(runLevel);
563             }
564 
565             worker.proceedTo(runLevel);
566         }
567     }

569     /**
570      * Get the best {@link Activator}.
571      *
572      * @return the best actvator; defaulting to this
573      */
574     protected synchronized Activator getActivator() {
575         Collection<Activator> activators = new ArrayList<Activator>();
576         List<ActiveDescriptor<?>> allActivators =
577                 serviceLocator.getDescriptors(BuilderHelper.createContractFilter(Activator.class.getName()));
578 
579         for (ActiveDescriptor<?> descriptor : allActivators) {
580             if (name.equals(Utilities.getRunLevelControllerName(descriptor))) {
581                 final Activator activator = (Activator) serviceLocator.getServiceHandle(descriptor).getService();
582                 if (activator != this) {
583                     activators.add(activator);
584                 }
585             }
586         }
587         return (activators.isEmpty()) ? this : activators.iterator().next();
588     }

629     private abstract class Worker implements Runnable {
630         /**
631          * The target run level.
632          */
633         protected volatile Integer planned;
634 
635         /**
636          * The active run level to be activated.
637          */
638         private Integer activeRunLevel;
639 
640         /**
641          * Tracks the direction of any active proceedTo worker.
642          */
643         protected Boolean upSide;
644 
645         /**
646          * Records whether a cancel was actually an hard interrupt.
647          */
648         protected Boolean isHardInterrupt;
649 
650 
651         // ----- Constructors -----------------------------------------------
652 
653         protected Worker(int runLevel) {
654             this.planned = runLevel;
655         }
656 
657 
658         // ----- Runnable ---------------------------------------------------
659 
660         /**
661          * Core control logic.
662          */
663         @Override
664         public void run() {
665             logger.log(LEVEL, "proceedTo({0}) - " + getDescription(true),
666                     planned);
667 
668             upSide = null;
669 
670             if (null != planned) {
671                 int current = getCurrentRunLevel();
672                 if (planned > current) {
673                     upSide = true;
674 
675                     int rl = current + 1;
676                     while (rl <= planned) {
677                         upActiveRecorder(rl);
678                         rl++;
679                     }
680                 } else if (planned < current) {
681                     upSide = false;
682 
683                     // start things off with a notification of the current
684                     // runLevel
685                     setCurrent(this, current);
686 
687                     down(current);
688                 } else { // planned == current
689                     upSide = false;
690 
691                     // force closure of any orphaned higher RunLevel services
692                     down(current + 1);
693                 }
694             }
695             finished(this);
696         }
697 
698         /**
699          * Attempts to interrupt processing to go to a new runLevel.
700          *
701          * @param isHard    if true, this was based on an explicit call to
702          *                  interrupt; false otherwise. The latter is the
703          *                  case for a new proceedTo() causing a cancel in
704          *                  order to proceedTo a new runLevel.
705          * @param runLevel  optionally, the revised runLevel to proceedTo
706          *
707          * @return true, if its possible to go to the new runLevel; note that
708          *         implementation may handle the interrupt by other means (i.e.,
709          *         throwing an InterruptException for the synchronous case)
710          */
711         public abstract boolean interrupt(boolean isHard, Integer runLevel);
712 
713         /**
714          * Called after initialization to run the proceedTo operation

788         private void activateRunLevel() {
789             List<ActiveDescriptor<?>> activations = serviceLocator.getDescriptors(
790                     new IndexedFilter() {
791                         private final String contract = RunLevel.class.getName();
792 
793                         @Override
794                         public boolean matches(Descriptor d) {
795                             return accept(d, activeRunLevel);
796                         }
797 
798                         @Override
799                         public String getAdvertisedContract() {
800                             return contract;
801                         }
802 
803                         @Override
804                         public String getName() {
805                             return null;
806                         }
807                     });
808 
809             Activator ia = getActivator();
810 
811             if (!activations.isEmpty()) {
812                 if (logger.isLoggable(LEVEL)) {
813                     logger.log(LEVEL, "sorting {0}", activations);
814                 }
815 
816                 Sorter sorter = getSorter();
817                 if (sorter != null) {
818                     sorter.sort(activations);
819                 }
820 
821                 for (ActiveDescriptor<?> descriptor : activations) {
822                     if (logger.isLoggable(LEVEL)) {
823                         logger.log(LEVEL, "activating {0} - "
824                                 + getDescription(true), descriptor);
825                     }
826 
827                     try {
828                         ia.activate(descriptor);
829                         // an escape hatch if we've been interrupted in some way
830                         checkInterrupt(null, descriptor, null);
831                     } catch (Exception e) {
832                         checkInterrupt(e, descriptor, null);
833                     }
834                 }
835             }
836             try {
837                 ia.awaitCompletion(DEFAULT_ASYNC_WAIT, TimeUnit.MILLISECONDS);
838             } catch (Exception e) {
839                 checkInterrupt(e, null, null);
840             }
841         }

843         private void deactivateRunLevel(int runLevel) {
844             List<Integer> downRecorders = getRecordersToRelease(runLevel);
845             for (int current : downRecorders) {
846                 Stack<ActiveDescriptor<?>> downRecorder;
847                 synchronized (lock) {
848                     downRecorder = recorders.get(current);
849                 }

851                 if (downRecorder != null) {
852 
853                     Activator ia = getActivator();
854 
855                     ActiveDescriptor<?> descriptor;
856                     while (!downRecorder.isEmpty()) {
857                         descriptor = downRecorder.pop();

860                                     + getDescription(true), descriptor);

866                             checkInterrupt(null, descriptor, null);
867                         } catch (Exception e) {

869                         }
870                     }

885     /**
886      * Sync worker
887      */
888     private class SyncProceedToWorker extends Worker {
889         // record the thread performing the operation
890         private final Thread activeThread = Thread.currentThread();
891 
892         // the next planned runLevel (after interrupt)
893         protected Integer nextPlannedAfterInterrupt;
894 
895         // records whether a cancel event was issued
896         private boolean cancelIssued;
897 
898         private SyncProceedToWorker(int runLevel) {
899             super(runLevel);
900         }
901 
902         /**
903          * Interrupts are always handled in the synchronous case either by
904          * popping the stack for the reentrant call on same thread, or by
905          * sending a cancel event to the active thread doing the proceedTo()
906          * call.
907          */
908         @Override
909         public boolean interrupt(boolean isHard, Integer runLevel) {
910             Thread ourThread = Thread.currentThread();
911 
912             synchronized (lock) {
913                 Integer planned = getPlannedRunLevel();
914                 if (!isHard && null != planned && planned.equals(runLevel)) {
915                     return true; // short circuit
916                 }
917 
918                 nextPlannedAfterInterrupt = runLevel;
919 
920                 if (ourThread == activeThread) {
921                     checkInterrupt(null, null, isHard);
922                 } else {
923                     // must interrupt another thread to do the new proceedTo().
924                     // Note how this thread exhibits async behavior in this
925                     // case.
926                     // The cancel notification will happen on the other thread
927                     logger.log(LEVEL, "Interrupting thread {0} - "
928                             + getDescription(true), activeThread);
929 
930                     this.isHardInterrupt = isHard;
931                     activeThread.interrupt();
932                 }
933             }
934 
935             return true;
936         }
937 
938         @Override
939         public void proceedTo(int runLevel) {
940             synchronized (lock) {
941                 planned = runLevel;
942                 nextPlannedAfterInterrupt = null;
943                 cancelIssued = false;
944                 isHardInterrupt = null;
945             }
946 
947             try {
948                 run();
949             } catch (Exception e) {
950                 handleInterruptException(e);
951             }
952         }
953 
954         @Override
955         protected void checkInterrupt(Exception e, ActiveDescriptor<?> descriptor,
956                                       Boolean isHard) {
957             synchronized (lock) {
958                 boolean cancelled = isCancelled(this);
959                 if (cancelled || null != nextPlannedAfterInterrupt) {
960                     if (!cancelled
961                             && canUpdateProceedTo(nextPlannedAfterInterrupt)) {
962                         planned = nextPlannedAfterInterrupt;
963                         nextPlannedAfterInterrupt = null;
964                         e = null;
965                     } else {
966                         // send cancel event, but only one time
967                         if (!cancelIssued) {
968                             cancelIssued = true;
969                             boolean wasHardInterrupt = isHardInterrupt(isHard,
970                                     e);
971                             isHardInterrupt = null;
972                             event(this, ListenerEvent.CANCEL,
973                                     null,
974                                     wasHardInterrupt);
975                         }
976 
977                         // pop stack to last proceedTo()
978                         throw new Interrupt();
979                     }
980                 }
981             }
982             super.checkInterrupt(e, descriptor, isHard);
983         }
984 
985         private boolean canUpdateProceedTo(Integer proposed) {
986             if (null != upSide) {
987                 Integer planned = getPlannedRunLevel();
988                 Integer active = getActivatingRunLevel();
989                 if (null != planned && null != active && null != proposed) {
990                     if (upSide && proposed > active) {
991                         return true;
992                     } else if (!upSide && proposed < active) {
993                         return true;
994                     }
995                 }
996             }
997 
998             return false;
999         }
1000 
1001         private void handleInterruptException(Exception e) {
1002             logger.log(LEVEL, "Interrupt caught - " + getDescription(true), e);
1003 
1004             Thread currentThread = Thread.currentThread();
1005 
1006             // we want to handle the new proceedTo if interrupted by another
1007             // thread,
1008             // otherwise we fall out since we are not the owning thread.
1009             Integer next = null;
1010             if (activeThread == currentThread) {
1011                 next = nextPlannedAfterInterrupt;
1012             }
1013 
1014             if (null != next) {
1015                 proceedTo(next);
1016             } else {
1017                 // RLS must continue / fall out
1018                 logger.log(LEVEL, "swallowing exception - "
1019                         + getDescription(true), new RunLevelException(e));
1020             }
1021         }

1039         /**
1040          * Interrupts are never handled in the asynchronous case.
1041          * <p/>
1042          * Here, we just kill the worker, and expect a new one to form.
1043          */
1044         @Override
1045         public boolean interrupt(boolean isHard, Integer runLevel) {
1046             boolean haveFuture;
1047             synchronized (lock) {
1048                 haveFuture = (null != activeFuture);
1049                 if (haveFuture) {
1050                     // cancel previous, but down hit thread with interrupt
1051                     activeFuture.cancel(false);
1052                     activeFuture = null;
1053                 }
1054             }
1055 
1056             if (haveFuture) {
1057                 event(this, ListenerEvent.CANCEL, null, isHard);
1058             }
1059 
1060             return false;
1061         }

1065             super.run();
1066             synchronized (lock) {

1068                 isHardInterrupt = null;
1069             }
1070         }
