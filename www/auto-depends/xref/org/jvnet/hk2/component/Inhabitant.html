50  /**
51   * Represents a component in the world of {@link Habitat}.
52   *
53   * <p>
54   * {@link Inhabitant} extends from {@link Holder}, as one of its
55   * purposes is to encapsulate how we obtain an instance of a component.
56   * On topf of that, {@link Inhabitant} enhances {@link Holder} by
57   * adding more metadata that {@link Habitat} uses for finding
58   * components and hooking them up together.
59   *
60   * <p>
61   * All the methods exept {@link #get()} are immutable, meaning
62   * they never change the value they return.
63   *
64   * <p>
65   * See {@link Inhabitants} for several factory methods for typical
66   * {@link Inhabitant} constructions.
67   * 
68   * @author Kohsuke Kawaguchi
69   * @see Inhabitants
70   */
71  // TODO: Eventually get rid of auto-depend's Holder
72  public interface Inhabitant<T> extends Binding<T>, ManagedComponentProvider<T>, Holder<T> {
73      /**
74       * The short-cut for {@code type().getName()}
75       * but this allows us to defer loading the actual types.
76       */
77      String typeName();
78  
79      /**
80       * Type of the inhabitant.
81       *
82       * <p>
83       * The only binding contract that needs to be honored is that the {@link #get()}
84       * method returns an instance assignable to this type. That is,
85       * {@code get().getClass()==type()} doesn't necessarily have to hold,
86       * but {@code type().isInstance(get())} must.
87       *
88       * <p>
89       * This is particularly true when {@link Factory} is involved, as in such
90       * case HK2 has no way of knowing the actual type.
91       *
92       * That said, this method is not designed for the semantics of
93       * contract/implementation split --- implementations of a contract
94       * should return the concrete type from this method, and use
95       * {@link Habitat#addIndex(Inhabitant, String, String) habitat index}
96       * to support look-up by contract. 
97       *
98       * @return
99       *      Always non-null, same value.
100      */
101     Class<? extends T> type();
102 
103     /**
104      * Returns the instance of this inhabitant.
105      *
106      * <p>
107      * Some {@link Inhabitant}s return the same instance for multiple
108      * invocations (AKA singleton), but
109      * the method may return different instances to invocations from different
110      * context (AKA scope.) The extreme case is where the each invocation
111      * returns a different object.
112      */
113     T get();
114 
115     /**
116      * Returns the instance of this inhabitant.
117      *
118      * <p>
119      * <b>THIS METHOD SHOULD BE ONLY USED BY HK2 IMPLEMENTATION</b>.
120      *
121      * <p>
122      * {@link Inhabitant}s are often used with the decorator pattern
123      * (see {@link com.sun.hk2.component.AbstractCreatorInhabitantImpl} for example), yet during
124      * the object initializtion inside the {@link #get()} method, we often
125      * need the reference to the outer-most {@link Inhabitant} registered to
126      * the {@link Habitat} (for example so that we can request the injection
127      * of {link Inhabita} that represents itself, or to inject companions.)
128      *
129      * <p>
130      * So this overloaded version of the get method takes the outer-most
131      * {@link Inhabitant}. This method is only invoked from within HK2
132      * where the decorator pattern is used.
133      */
134     // TODO: this and the lead/companions method make you wonder whether we should
135     // define Inhabitant as an abstract class.
136     T get(Inhabitant onBehalfOf);
137 
138     /**
139      * Gets the metadata associated with this inhabitant.
140      *
141      * <p>
142      * This data is usually used by a sub-system of HK2, and not really meant to
143      * be used by applications. (At least for now.)
144      * The main benefit of metadata is that it's available right away
145      * as soon as the {@link Habitat} is properly initialized, even before
146      * component classes are loaded. In contrast, accessing annotations would require
147      * classes to be loaded and resolved.
148      *
149      * @return
150      *      can be empty but never null. The values are read-only.
151      * @see Service#metadata() 
152      */
153     MultiMap<String,String> metadata();
154 
155     /**
156      * Obtains the serialized metadata.
157      *
158      * <p>
159      * This method is a wrapper around {@link #metadata()} and useful for
160      * defining a highly structured metadata that doesn't easily fit
161      * a simple string representation.
162      *
163      * <p>
164      * The implementation of this method is to obtain the value associated with
165      * this key as {@code metadata().getOne(key)}, and if that exists, treat
166      * the value as base64-encoded binary, and deserializes it and returns the object.
167      *
168      * <p>
169      * The classes used in the serialization need to be available during the build time
170      * (normally during the HK2 compile mojo runs) so that the metadata can be
171      * serialized. The evolution of these classes need to be careful done, otherwise
172      * the deserialization of the metadata may fail unexpectedly.
173      *
174      * @throws Error
175      *      If the deserialization fails. This can be for example because of
176      *      the incompatible class change, or failure to resolve the classes.
177      *      Sine these problems can only happen in a critical situation,
178      *      this method throws unchecked error.
179      *      TODO: switch this to IOError when we can depend on JDK6.
180      *
181      * @return
182      *      the deserialized object.
183      */
184     <T> T getSerializedMetadata(Class<T> type, String key);
185 
186     /**
187      * Obtains the metadata serialized into String.
188      *
189      * <p>
190      * This is a convenient short-cut that does {@code getSerializedMetadata(type,type.getName())}
191      */
192     <T> T getSerializedMetadata(Class<T> type);
193 
194     /**
195      * Called to orderly shutdown {@link Habitat}.
196      * <p>
197      * The expected behavior is for objects to get its {@link org.glassfish.hk2.PreDestroy}
198      * callback invoked, and its reference released. For singleton
199      * objects, this method is expected to dispose that object.
200      * <p>
201      * For scoped objects, those are released when {@link org.glassfish.hk2.ScopeInstance#release()}
202      * is invoked.
203      */
204     void release();
