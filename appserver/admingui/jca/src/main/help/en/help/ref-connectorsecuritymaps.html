Connector Connection Pool Security Maps

Use the Connector Connection Pool Security Maps page to manage security maps for connector connection pools.

Security maps for connector connection pools enable the creation of a mapping between Java EE users and groups and EIS users and groups. A security map maps the principal received during servlet or EJB authentication to the credentials accepted by the EIS. It is possible to map multiple GlassFish Server principals to the same back-end principal.

A security map for a connection pool is different from a connector work security map, which maps a principal associated with an incoming work instance to a principal in a security realm of the GlassFish Server domain. See Work Security Maps for information about connector work security maps.

For each security map, the following information is provided.

Name

The name of the security map.

The Security Maps table also contains the following options.

New

Button to create a new security map.

Delete

Button to delete one or more selected security maps.
