To Test if a Node Is Reachable

To create or start GlassFish Server instances on a node that represents a remote host, the DAS must be able to contact the node. Before attempting to create or start instances on such a node, test that the DAS can contact the node.

The type of the node that you are testing must be SSH. You cannot test a node of type CONFIG because a node of type CONFIG is not enabled for remote communication.

Before You Begin

Ensure that the appropriate protocol for the type of the node is configured on the host that the node represents. The node is enabled for remote communication over this protocol.

  • If the type of the node is SSH, the secure shell (SSH) protocol must be configured.

    1. In the navigation tree, select the Nodes node.

      The Nodes page opens. This page lists all nodes in the current domain.

    2. In the table of nodes, follow the Ping link for the node that you are testing.

      A message that indicates whether the DAS could contact the node is added to the Nodes page.
